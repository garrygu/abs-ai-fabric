## 1. 项目目的

**ABS AI Fabric** 是一套运行在 ABS 工作站上的**本地 AI 平台**，目标是在不依赖公有云的前提下，为企业和演示场景提供可管理的 AI 能力。

核心价值：

- **数据不出站**：推理、向量检索、文档处理均在本地完成
- **统一管控**：通过 Hub Gateway 和 AI Fabric 管理界面，集中管理模型、服务与应用
- **可扩展**：新应用通过 `app.manifest.json` 和 Gateway API 接入，不直接绑定底层实现
- **演示就绪**：Workstation Console 提供 GPU 可视化与 CES 展示模式

典型应用场景包括：合同审阅（Contract Reviewer）、法律助手、Onyx 对话/RAG、系统可观测性展示等。

## 2. 架构概览

### 2.1 分层结构

```
用户 / 浏览器
    ↓
应用层（Vue 前端 + Docker 化 Python 应用）
    ↓
Hub Gateway (:8081) — OpenAI 兼容 API、资产注册、服务编排、Auto-Wake
    ↓
核心服务（Docker，共享 abs-net 网络）
    ↓
硬件（NVIDIA GPU + 本地存储）
```

### 2.2 设计原则（接手后务必理解）

| 原则 | 说明 |
|------|------|
| **一切皆 Asset** | 模型、向量库、缓存、工具均在 `assets/` 注册，由 Gateway 统一管理生命周期 |
| **接口驱动** | 应用声明所需接口（如 `llm-runtime:v1`），不硬编码 Ollama/Qdrant 等实现 |
| **Core 可演进** | Ollama 可替换为 vLLM，Qdrant 可替换为其他向量库，通过 `bindings.yaml` 切换 |
| **Auto-Wake** | 按需启动高成本服务，空闲后自动休眠，降低 GPU/内存占用 |

### 2.3 核心服务一览

| 组件 | 端口 | 容器名 | 职责 |
|------|------|--------|------|
| Hub Gateway | 8081 | `abs-core-hub-gateway` | API 门面、资产目录、服务控制 |
| Ollama | 11434 | `abs-ollama` | 本地 LLM 推理 |
| Qdrant | 6333 | `abs-qdrant` | 向量检索 |
| Redis | 6379 | `abs-redis` | 缓存与会话 |
| PostgreSQL | 5432 | `document-hub-postgres` | 文档元数据（Document Hub） |
| GPU Metrics | 8083 | 宿主机 Node 进程 | GPU 监控（由 `start-core.ps1` 启动） |

### 2.4 主要应用

| 应用 | 路径 | 端口 | 运行方式 |
|------|------|------|----------|
| AI Fabric（管理 UI） | `abs-ai-hub/hub-ui-v2` | 5173 | `npm run dev` |
| Workstation Console | `abs-ai-hub/apps/abs_workstation_console` | 5200 | `npm run dev` |
| Contract Reviewer v2 | `abs-ai-hub/apps/contract-reviewer-v2` | 8082 | Docker Compose |
| Legal Assistant | `abs-ai-hub/apps/legal-assistant` | 8050 | Docker Compose |
| Onyx Suite | `abs-ai-hub/apps/onyx-suite` | 8150 | Docker Compose |
| Whisper Server | `abs-ai-hub/apps/whisper-server` | 8001 | Docker Compose |

应用清单注册在 `abs-ai-hub/apps-registry.json`；各应用另有 `app.manifest.json`。

## 4. 环境与启动（最小路径）

### 4.1 前置条件

- Windows 10/11 + Docker Desktop（WSL2）
- NVIDIA 驱动 + CUDA（LLM 推理）
- Node.js 18+（Hub UI、Workstation Console、GPU Metrics）
- PowerShell（运行 `core/*.ps1` 脚本）

### 4.2 标准启动顺序

```powershell
# 1. 核心服务
cd core
copy .env.example .env    # 首次
.\start-core.ps1

# 2. AI Fabric 管理界面
cd ..\abs-ai-hub\hub-ui-v2
npm install && npm run dev    # http://localhost:5173

# 3. （可选）Workstation Console
cd ..\apps\abs_workstation_console
npm install && npm run dev    # http://localhost:5200
```

Windows 也可直接运行仓库根目录的 `Start.bat`。

验证 Gateway：`curl http://localhost:8081/health`

## 6. 注意事项与已知情况

### 6.1 平台级

1. **Gateway 是单一入口**  
   应用应调用 `http://hub-gateway:8081`（容器内）或 `http://localhost:8081`（宿主机），不要绕过 Gateway 直连 Ollama/Qdrant，否则 Auto-Wake 和资产治理会失效。

2. **修改 Gateway 代码必须 rebuild 镜像**  
   `core/gateway/` 下 Python 变更后需 `docker compose build hub-gateway`，仅 `docker restart` 不够。详见 [RESTART_INSTRUCTIONS.md](../RESTART_INSTRUCTIONS.md)。

3. **Hub UI 不是 Docker 容器**  
   `hub-ui-v2` 和 `abs_workstation_console` 均为 Vite 开发服务器；生产部署需 `npm run build` 后自行托管静态资源。

4. **GPU Metrics 运行在宿主机**  
   由 `start-core.ps1` 通过 PowerShell Job 启动 Node 进程（:8083），依赖本机安装 Node.js；Docker 内 Gateway 无法直接访问 GPU，需此 sidecar。

5. **网络名 `abs-net`**  
   所有容器化应用必须在 compose 中加入同一 Docker 网络，服务间用容器名通信。

6. **`apps-registry.json` 与目录不完全同步**  
   注册表仍含部分历史条目（如 `rag-pdf-voice`、旧版 contract-reviewer）；实际 `apps/` 目录以 manifest 和 compose 文件为准，接手后建议做一次 registry 清理。

### 6.2 配置与安全

- `core/.env` 从 `.env.example` 复制，含网络名、端口等；**勿提交含密钥的 `.env`**
- PostgreSQL 默认密码在 `docker-compose.yml` 中为示例值（`secure_password`），**生产环境必须更换**
- CES 模式下 Admin 操作为只读，通过 `VITE_CES_MODE=true` 或 `CES_MODE=true` 启用

### 6.3 开发债务 / 待关注

- Contract Reviewer v2 部分分析逻辑仍有 TODO（见 `contract_reviewer_persistence.py`）
- `apps-registry.json` 中 Contract Reviewer v2 端口为 8082（host 映射），manifest 内写的是容器端口 8080，查阅时注意区分
- 部分文档路径、脚本中的 `C:\ABS\` 或 `D:\abs-ai-fabric\` 为历史路径，以仓库相对路径为准
- 根目录 `LICENSE` 尚未填写

### 6.4 常见排障

| 现象 | 优先检查 |
|------|----------|
| Gateway 503 / Store 不可用 | `docker logs abs-core-hub-gateway`，确认 `store-sources.json` 挂载 |
| UI 无法加载数据 | Gateway `/health`；Vite proxy 是否指向 :8081 |
| 应用连不上 Core | 是否在 `abs-net`；是否用了 `localhost` 而非服务名 |
| GPU 指标缺失 | 本机是否安装 Node；`Get-Job GPUMetricsServer` |
